# roadmap-orchestrator

Агентский скилл, превращающий основную сессию в чистого оркестратора конвейера:
**roadmap → spec → plan → implementation → code review**

Каждый артефакт пишется заспавненным агентом. Между этапами — кросс-модельные волны ревью (ревьюер ≠ писатель), пока блокирующих находок не останется. Оркестратор сам не пишет содержимое артефактов — только ведёт диалог и учитывает состояние конвейера.

## Как работает конвейер

```
ROADMAP.md
  └── выбрать следующий незавершённый этап
        │
        ▼
  ┌─────────────────────────────────────────────┐
  │  Этап спеки                                 │
  │  1. Оркестратор: диалог с пользователем     │
  │  2. Спавн писателя → документ спеки         │
  │  3. Кросс-модельная волна ревью             │
  │     ├── блокеры → спавн фиксера → повтор    │
  │     └── только Minor → GATE PASS            │
  └─────────────────────────────────────────────┘
        │
        ▼
  ┌─────────────────────────────────────────────┐
  │  Этап плана                                 │
  │  1. Спавн писателя → документ плана         │
  │  2. Кросс-модельная волна ревью             │
  │     ├── блокеры → спавн фиксера → повтор    │
  │     └── только Minor → GATE PASS            │
  └─────────────────────────────────────────────┘
        │
        ▼
  ┌─────────────────────────────────────────────┐
  │  Реализация                                 │
  │  superpowers:subagent-driven-development     │
  │  пишущая модель по рубрике маршрутизации    │
  └─────────────────────────────────────────────┘
        │
        ▼
  ┌─────────────────────────────────────────────┐
  │  Ревью кода (весь дифф этапа BASE..HEAD)    │
  │  ревьюер ≠ модель реализатора               │
  │     ├── блокеры → спавн фиксера → повтор    │
  │     └── только Minor → GATE PASS            │
  └─────────────────────────────────────────────┘
        │
        ▼
  следующий пункт ROADMAP.md
```

**Три жёстких правила:**
1. **Оркестратор не пишет** — содержимое спеки, плана и кода только в заспавненных агентах
2. **Кросс-модельное ревью** — ревьюер всегда другая модель/CLI, чем писатель
3. **Волны до нуля блокеров** — конвейер не движется, пока Critical/Important-находки не закрыты

## Предварительные требования

### Обязательно

- **superpowers** (плагин Claude Code, v5.1.0+) — установлен и включён в `~/.claude/settings.json`
  ```bash
  # Проверить, что плагин активен (не стоит в false):
  cat ~/.claude/settings.json | grep -A2 superpowers
  ```
- **codex** (codex-cli, v0.137.0+) — для кросс-модельного ревью
  ```bash
  command -v codex && codex --version
  ```

### Для запасного режима (Codex-оркестратор + DeepSeek-ревьюер)

- **deepcode** (`@vegamo/deepcode-cli`, v0.1.29+) или ключ DeepSeek API в `~/.deepcode/settings.json`
  ```bash
  command -v deepcode
  cat ~/.deepcode/settings.json  # должен содержать api_key
  ```
- `~/.agents/skills/superpowers/` должен резолвиться:
  ```bash
  ls ~/.agents/skills/superpowers/SKILL.md 2>/dev/null || echo "Нет — создать симлинк:"
  # ln -s ~/.claude/plugins/cache/claude-plugins-official/superpowers/5.1.0 ~/.agents/skills/superpowers
  ```

## Установка

```bash
git clone https://github.com/YOUR_USERNAME/skill-coding.git
cd skill-coding
bash install.sh
```

`install.sh` идемпотентно создаёт два симлинка:
- `~/.claude/skills/roadmap-orchestrator` → `<клон>/skills/roadmap-orchestrator`
- `~/.agents/skills/roadmap-orchestrator` → `<клон>/skills/roadmap-orchestrator`

Проверьте, что симлинк указывает на **ваш клон** (не скачанный zip):
```bash
ls -la ~/.claude/skills/roadmap-orchestrator
```

## Использование

В новой сессии Claude Code:
```
работаем по роадмапу
```
или
```
запусти конвейер
```

Скилл найдёт `ROADMAP.md` (или спросит путь), выберет следующий незавершённый этап и начнёт конвейер.

## Границы данных — важно прочитать перед использованием

Этот скилл отправляет содержимое вашего репозитория во внешние API:

- **Волны ревью через `codex exec`** — уходят в OpenAI API (модели Codex/GPT)
- **Запасной режим через DeepSeek** — уходят в DeepSeek API
- **Каждая волна** включает диффы, спеки, планы или пути к файлам репозитория

**Не используйте этот скилл на репозиториях, содержащих:**
- API-ключи, токены, пароли (даже в комментариях)
- PII (персональные данные пользователей)
- Проприетарный код под NDA
- Секреты инфраструктуры

`.gitignore` исключает `settings.json` и `.env*`, но он не защищает от случайного ввода секретов в промпты.

## Структура репозитория

```
skill-coding/
  README.md                      — этот файл
  install.sh                     — установка симлинков
  skills/
    roadmap-orchestrator/
      SKILL.md                   — входная точка скилла
      references/
        review-protocol.md       — цикл волн, контракт находок, критерий остановки
        model-routing.md         — рубрика маршрутизации моделей
        spawn-recipes.md         — рецепты спавна под каждый harness
        fallback-mode.md         — запасной режим Codex+DeepSeek
        prompts/
          spec-review.md         — нейтральный промпт ревью спеки
          plan-review.md         — нейтральный промпт ревью плана
          code-review.md         — нейтральный промпт ревью кода
  docs/
    plans/                       — планы реализации
```
